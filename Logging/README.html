README for mobius.logging

Welcome to mobius.logging, an advanced debugging framework for Java!

Source Code

This release of mobius.logging does include source code. See the source directory for more information.

If you wish to contribute to mobius.logging, please email mobius.logging@kindsoftware.com and we can discuss our process (e.g. patches, CVS access, contributor maintaining ownership of additions/changes, etc.).

Documentation

This release of mobius.logging comes with several sets of documentation.

  1. The mobius.logging brief, a short summary of the framework.
  2. A design overview for a top-level perspective of the system. This document includes system requirements, background, and a design dictionary.
  3. A full BON specification of mobius.logging. See ISE's BON summary for more information on BON.
    Note that in the design overview and BON specification we call the general system being specified the "Monitoring" system. mobius.logging is an implementation of this specification.
  4. Full Javadoc-generated documentation is included in the docs/javadoc directory.
  5. A technical report describing mobius.logging is included in the directory docs/manual. This document isn't a true "user's manual". Such a document will be included in the 1.0 release of mobius.logging.
  6. Miscellaneous other documentation (release notes, FAQ, TODO list, etc.) are included in several text files in the same directory as this file.
  7. Use the source! The source code, or more specifically, the full Design by Contract specification that can be found in the source, is your best guide toward the correct use of mobius.logging. We, of course, use the excellent Infospheres Java Code Standard in mobius.logging.

Using mobius.logging

To learn how to use mobius.logging, one should read the mobius.logging technical report and peruse the Javadoc documentation. The BON specification is also quite useful in getting a more top-level view of the architecture. Additionally, the source code for the mobius.logging blackbox test suite has been included in the source/mobius.logging/testsuite directory, which should be very revealing to the interested developer. In that same directory one can find an example extension to the DebugConstants interface called FrenchConstants.

Simply add the mobius.logging jar file to your CLASSPATH to use mobius.logging.
